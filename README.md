# Annual Key Tracker — Update v3

Upload every file in this folder to the root of your existing GitHub repository. Allow GitHub to replace the files with the same names.

## Included in this update

- Simplified top bar with the menu in the upper-left corner.
- Daily and Weekly selector in the main view.
- The calendar opens only when the displayed date or week field is tapped.
- Go to Today appears only while another date or week is selected.
- Dashboard, Reports, Custom Range, Settings, and Backup & Restore are inside the menu.
- Daily Summary is placed above Activity Entry.
- Daily Summary shows Total Interactions, Contacted, Snoozed, and Unable to Contact.
- PPQ was replaced with Outcomes: PA PPQ or Appeals PPQ.
- The selected Outcomes workflow is saved on each individual activity entry, so switching workflows during a day carries into reports.
- A new day begins with the previous workday’s Key Tracker or Tally Counter method ready. The selector disappears when the agent switches methods or records the first activity.
- No tracking-method label remains after the selection is locked.
- Day Status is removed from today’s activity screen. Previous and future dates can be marked Scheduled Off, PTO, Holiday, or Leave.
- A past weekday with no activity and no selected reason automatically shows as Absent.
- Absent and non-working dates are excluded from averages and goals.
- Weekly and Custom Range views always ignore weekends and saved holidays.
- Key Tracker groups receive one timestamp and appear in the Daily Timeline and Interactions Recorded by Hour chart.
- Tally Counter additions remain untimestamped.
- Automatic device time-zone detection remains available with a manual time-zone setting.

## GitHub upload

1. Open the repository.
2. Choose **Add file → Upload files**.
3. Select all eight files from this folder.
4. Use the commit message: `Update tracker layout and Outcomes workflows`
5. Choose **Commit changes**.
6. Wait about a minute, then refresh the GitHub Pages website. If the older layout remains, fully close the browser tab and reopen the site.

Tracker data remains in the browser’s local storage. Replacing these website files does not intentionally erase existing saved data.
